# 醒了吗? App Store Readiness

更新时间：2026-03-31  
适用目录：`/Users/howell623/Projects/醒了吗?`

## 当前结论

当前版本 **还不适合直接上架 App Store 开始赚钱**。  
原因不是算法或页面完全不可用，而是“可售卖、可审核、可交付”的关键链路还没全部闭环。

## 已达到

- [x] iOS 原生主工程可构建
- [x] 核心酒精算法已有 Swift 单测
- [x] StoreKit 2 已有真实结构壳
- [x] 购买成功后已补 `transaction.finish()`
- [x] 恢复购买入口已接通
- [x] 通知权限申请与本地提醒调度已落地基础版本
- [x] 历史页、设置页、升级页已存在
- [x] 中英文本地化已具备基础骨架

## 阻塞项

- [ ] App Store Connect 真实内购商品尚未接通并完成真机验证
- [ ] App Icon 资源未提供，当前只有 `Contents.json`，没有实际图片文件
- [ ] 缺少 App Store 上架素材：截图、描述、关键词、推广文案
- [ ] 缺少隐私政策与支持页面 URL
- [ ] 缺少完整的付费验证记录：购买、恢复购买、异常降级、退款后状态
- [ ] 守候闭环仍停留在本地通知提醒，没有真正的多联系人兜底流程
- [ ] 缺少一轮面向真实用户的体验收口：首次引导、权限解释、异常态说明
- [ ] 尚未完成真机环境下的通知触发验证
- [ ] 尚未完成至少一次归档/历史/清空/跨天回滚的真机验收

## 建议上线前顺序

1. 接通 App Store Connect 商品，完成 Sandbox 真机购买与恢复购买。
2. 补齐 App Icon、截图、隐私政策 URL、支持页 URL。
3. 完成真机通知验证，确认锁屏和后台下的守候提醒行为。
4. 冻结 Free / Pro 权益边界，避免审核前后文案和功能不一致。
5. 做一轮 TestFlight 收口，修掉上手阻力和误解点后再提审。

## 备注

- 当前更像“可继续打磨并进入 TestFlight 的候选版”，不是“今天就能直接提审变现”的完成版。
- 如果只看代码质量和基础功能，已经明显高于最早的 Web MVP。
- 如果按“能不能开始稳定赚钱”来衡量，仍缺少商品、素材、合规和真机验证这四块。
