# 「醒了吗」项目交接文档

更新时间：2026-03-20  
当前版本：Web MVP / 本地静态前端版

## 1. 项目概述

### 1.1 产品定位

「醒了吗」是一个极简饮酒安全兜底与清醒计时工具。核心目标不是教育用户“少喝酒”，而是在酒局中提供：

- 当前饮酒量的即时感知
- 距离清醒还有多久的粗略估算
- 超量后的静默巡逻与提醒兜底
- 尽量低打扰、低操作负担的记录方式

### 1.2 当前实现形态

当前交付的是纯前端 Web MVP：

- 无后端
- 无数据库
- 无账户系统
- 所有数据仅保存在浏览器 `localStorage`
- 可以通过静态服务器直接运行

当前版本适合：

- 产品验证
- 交互演示
- 视觉方案迭代
- 原生 App 开发前的逻辑确认

当前版本不适合直接作为最终生产形态上线，原因见“限制与风险”章节。

## 2. 当前完成内容

### 2.1 已完成的业务能力

- 首次配置：
  - 性别
  - 体重
  - 个人酒量阈值（按纯酒精克数）
  - 紧急联系人手机号
- 酒局会话管理：
  - 今日酒局状态记录
  - 跨天自动清空当局
  - 点击“我已安全 / 结束酒局”后立即停掉巡逻
- 饮酒记录：
  - 啤酒 / 红酒 / 白酒 / 洋酒四类
  - 每类支持 `+` / `-`
  - 首次记录某类酒时支持设置本次酒局的度数和单次容量
- 计算逻辑：
  - 纯酒精克数计算
  - 清醒倒计时
  - 预计清醒时刻
  - 各酒类对总饮酒量的占比
- 安全机制：
  - 达到阈值 50% 触发一次平安报备
  - 超阈值 30 分钟无操作触发预警拦截
  - 超阈值 60 分钟无操作触发紧急拦截
  - 任意加减酒操作重置巡逻计时
- Web 可实现的短信兜底：
  - 预填短信内容
  - 尝试唤起系统短信页
  - 同时复制短信文案
- Demo 能力：
  - 一键载入演示酒局

### 2.2 已完成的视觉方向

目前主界面已改成接近“市场面板 / 数据行情看板”的视觉语言，参考方向类似 Polymarket：

- 顶部导航 + 搜索条 + 操作按钮
- 主卡片展示本局核心状态
- 右侧信息栏展示醉酒度、巡逻状态、最近提醒
- 饮酒项展示为“饮酒市场”
- 整体颜色根据醉酒程度联动

### 2.3 醉酒度联动色

当前整页主题色按阈值占比切换：

- 蓝：未达到 50%，视为清醒
- 黄：达到 50%，视为微醺
- 橙：达到 100%，视为醉酒
- 红：达到 150%，视为高危

这部分是页面级主题联动，不只是状态标签变色。

## 4. 当前核心逻辑

### 4.1 数据分层

#### 持久化配置

存储在 `localStorage` 的 `wake-app-config`：

- `gender`
- `weight`
- `alcoholThreshold`
- `emergencyContact`

#### 今日酒局会话

存储在 `localStorage` 的 `wake-app-session`：

- `dateKey`
- `startedAt`
- `lastDrinkTime`
- `safeMode`
- `safeReportSent`
- `warningHandled`
- `emergencyHandled`
- `messageLog`
- `drinks`

其中 `drinks` 按酒类保存：

- `count`
- `abv`
- `volumeMl`

### 4.2 饮酒计算公式

当前实现公式：

- 纯酒精 `g = 容量(ml) × 度数(%) × 0.8`
- 清醒时长 `h = 纯酒精(g) ÷ 8g/h`
- 预计清醒时刻 `= 最后一次饮酒操作时间 + 清醒时长`

说明：

- 当前没有做男女、体重、个体代谢差异的修正
- 体重、性别已录入，但暂未参与代谢公式
- 这是当前版本最容易继续升级的点之一

### 4.3 巡逻与提醒逻辑

#### 平安报备

触发条件：

- 当前总纯酒精量 `>= 阈值 50%`
- 本局尚未发送过平安报备

行为：

- 记录到 `messageLog`
- 生成平安报备文案
- 复制短信内容
- 尝试唤起短信页

#### 饮酒预警

触发条件：

- 当前总纯酒精量 `>= 阈值`
- 连续 30 分钟没有加减酒操作

行为：

- 本地弹出提醒面板
- 显示 60 秒倒计时
- 触发振动
- 用户可以取消本次提醒
- 若未取消，则尝试打开短信页

#### 紧急提醒

触发条件：

- 当前总纯酒精量 `>= 阈值`
- 连续 60 分钟没有加减酒操作

行为：

- 本地弹出更高等级提醒
- 倒计时 60 秒
- 强提醒振动
- 若未取消，则尝试打开短信页

#### 重置条件

以下行为会重置巡逻：

- 任意 `+` / `-` 操作
- 点击“我已安全 / 结束酒局”
- 清空酒局
- 跨天进入新的一天

## 9. 当前限制与风险

### 9.1 Web 版天然限制

这是当前版本最大的现实边界：

- 浏览器不能真正后台常驻巡逻
- 页面被系统杀掉、浏览器被关闭后，计时器会停止
- 浏览器不能静默自动发送短信
- 目前只能“尝试唤起短信页 + 复制短信内容”

结论：

如果要把“关键时刻兜底”做成真正可用产品，必须继续往原生 App 走。

### 9.2 公式仍是简化版

当前公式只按：

- 摄入纯酒精
- 固定 8g/h 代谢速率

未考虑：

- 性别代谢差异
- 体重修正
- 饮酒速度影响
- 空腹 / 饭后差异
- 个体差异

因此当前仅适合作为风险提示，不适合作为医学判断。

### 9.3 视觉图表是示意型

当前主卡片内的柱状趋势图是“状态可视化示意”，不是历史真实采样曲线。  
如果后续要做真实曲线，需要增加：

- 操作时间点采样
- 每次变动后的状态快照
- 真实时间序列渲染
