# 「醒了吗」产品需求文档

版本：V1.0
更新时间：2026-03-20
文档类型：产品交付版

## 1. 产品定义

### 1.1 产品名称

「醒了吗」

### 1.2 产品定位

一个极简、低打扰的饮酒安全兜底与清醒计时工具。

### 1.3 核心目标

用户在酒局中只需要极少操作，就能快速获得：

- 当前累计饮酒强度
- 距离清醒还有多久
- 是否接近或超过个人酒量阈值
- 超量后是否进入静默巡逻
- 关键时刻是否有兜底提醒

### 1.4 非目标

本产品不追求：

- 戒酒教育
- 卡路里统计
- 历史打卡
- 社区互动
- 复杂报表

## 2. 目标用户

### 2.1 核心用户

25-60 岁，有应酬、聚餐、社交饮酒场景的成年人。

### 2.2 用户特征

- 酒局中不愿做复杂输入
- 不喜欢说教型饮酒工具
- 更在意“多久能醒”和“出事时有人知道”
- 对隐私、体面和低打扰有较高要求

## 3. 核心价值

### 3.1 即时判断

让用户快速知道当前饮酒量和大致清醒时间。

### 3.2 静默兜底

在用户超量且长时间无操作时，提供提醒和联系人兜底机制。

### 3.3 低负担

操作尽量简化为：

- 点一次加酒
- 点一次撤销
- 必要时点“我已安全”

## 4. 核心原则

### 4.1 零牵挂

- 数据本地保存
- 今日酒局结束即清空
- 不做复杂历史负担

### 4.2 不打扰兴致

- 主流程简单
- 信息展示直观
- 提醒只在关键时刻出现

### 4.3 体面优先

- 对外文案避免“喝酒出事”直白措辞
- 以“应酬”“状态提醒”等表述替代

### 4.4 先拦截，再触发

- 所有预警和紧急提醒都必须先给用户本地倒计时拦截机会

## 5. 功能范围

### 5.1 首次配置

首次进入必须填写：

- 性别
- 体重（kg）
- 个人酒量阈值（按纯酒精克数）
- 紧急联系人手机号

说明：

- 当前版本里体重、性别已录入，但代谢公式暂未使用
- 阈值是当前产品最重要的个体化参数

### 5.2 酒局记录

支持记录四类酒：

- 啤酒
- 红酒
- 白酒
- 洋酒

每类酒支持：

- `+` 加一份
- `-` 撤销一份

首次记录某类酒时：

- 可修改默认酒精度
- 可修改默认单次容量
- 本次酒局中后续沿用

### 5.3 核心结果展示

主界面需展示：

- 累计纯酒精克数
- 阈值占比
- 清醒倒计时
- 预计清醒时刻
- 当前状态标签
- 饮酒构成占比
- 巡逻状态

### 5.4 安全机制

#### 平安报备

触发条件：

- 达到个人阈值 50%
- 本局未发送过平安报备

效果：

- 生成平安报备短信文案
- 记录在巡逻日志中

#### 饮酒预警

触发条件：

- 达到或超过个人阈值
- 连续 30 分钟无加减酒操作

效果：

- 弹出本地提醒
- 60 秒倒计时
- 振动提醒
- 用户可取消
- 超时后触发短信兜底动作

#### 紧急提醒

触发条件：

- 达到或超过个人阈值
- 连续 60 分钟无加减酒操作

效果：

- 更高等级提醒
- 60 秒倒计时
- 强振动
- 超时后触发短信兜底动作

### 5.5 结束酒局

主界面底部固定按钮：

- “我已安全 / 结束酒局”

点击后效果：

- 停止所有巡逻
- 停止所有提醒队列
- 当前酒局视为结束

## 6. 数据规则

### 6.1 持久化数据

本地持久保存：

- 性别
- 体重
- 酒量阈值
- 紧急联系人

### 6.2 会话数据

仅保存当天酒局：

- 酒类记录
- 最后操作时间
- 巡逻状态
- 是否已发送报备
- 提醒日志

### 6.3 清理规则

以下情况清空当局：

- 用户点击“我已安全 / 结束酒局”并重新开始新一局
- 用户手动清空本局
- 日期跨天

## 7. 计算规则

### 7.1 酒精量计算

公式：

```text
纯酒精(g) = 容量(ml) × 度数(%) × 0.8
```

### 7.2 清醒时间计算

公式：

```text
清醒时长(h) = 纯酒精(g) ÷ 8g/h
```

### 7.3 预计清醒时刻

公式：

```text
预计清醒时刻 = 最后一次饮酒操作时间 + 清醒时长
```

### 7.4 醉酒度等级

当前按阈值占比分层：

- `< 50%`：清醒
- `>= 50%`：微醺
- `>= 100%`：醉酒
- `>= 150%`：高危

## 8. 文案规范

### 8.1 平安报备

```text
【平安报备】我正在应酬，当前饮用：XXX，状态安全，无需担心。
```

### 8.2 饮酒预警

```text
【饮酒预警】亲友已超量，当前饮用：XXX，处于醉酒状态，请提醒休息。
```

### 8.3 紧急提醒

```text
【紧急提醒】亲友醉酒后长时间无操作，当前饮用：XXX，建议关注其安全。
```

## 9. 视觉方向

### 9.1 当前方向

当前版本采用“市场 / 行情面板”视觉语言：

- 顶部导航
- 数据主卡片
- 趋势图
- 右侧信息栏
- 大量结构化信息面板

### 9.2 色彩联动

颜色跟随风险等级切换：

- 蓝：清醒
- 黄：微醺
- 橙：醉酒
- 红：高危

### 9.3 设计目标

- 第一眼像实时面板，而不是教育工具
- 状态变化必须清晰
- 黄橙红切换要让用户立刻感知风险升级

## 10. 成功标准

MVP 阶段重点验证：

- 用户是否能快速完成首次设置
- 是否能低成本记录酒局
- 是否能看懂当前风险等级
- 是否能理解清醒倒计时
- 是否能接受提醒和兜底机制

## 11. 当前版本边界

### 11.1 当前是 Web MVP

这意味着：

- 无法后台常驻
- 无法静默自动发短信
- 页面被关闭后巡逻停止

### 11.2 当前不是最终生产版

如果要做成真实可上线安全工具，必须继续往原生 App 演进。

## 12. 后续建议

建议后续按两个方向推进：

- 产品逻辑升级：
  - 更完整的 Widmark / BAC 模型
  - 体重与性别入公式
- 技术形态升级：
  - 原生化
  - 更强的本地提醒与后台能力
